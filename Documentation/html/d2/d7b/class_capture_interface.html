Detailed Description

The interface to be used by all video capture methods.

Author
Stefan Zickler, (C) 2008

The CaptureInterface is the base class that should be inherited for any particular capturing method implementation.

It relies on the VarTypes system (see

See also
VarTypes.h) for the management an introspection-abilities of settings.

bool CaptureInterface::copyAndConvertFrame ( const RawImage &  src,
RawImage &  target 
)
virtual

This function will allow the copying of a captured frame to another RawImage data-structure. Overloading this function is recommended to provide more advanced functionality, such as converting the frame from one format to another. which format to convert to, should be a VarTypes option, added to the settings above. For an example implementation, please see how this function is overloaded in the CaptureDC1394v2 class.

In its current implementation in this base-interface, all this function will do is allocate the target image (if not already allocated, and then memcpy the data as-is.

Reimplemented in CaptureDC1394v2, CaptureV4L, CaptureBlueFox2, CaptureFlycap, CaptureFromFile, and CaptureGenerator.

bool CaptureInterface::resetBus ( )
virtual

If applicable, this will enforce a reinitialization / reset on your capturing hardware, in case something went wrong.

This is applicable, for example for DC1394 based systems where a firewire bus is able to 'freeze up' and can be reinitialized by a bus reset.
