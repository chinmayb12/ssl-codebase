1 # GrSim - INSTALL
2 
3 ## Overview
4 
5 We developed grSim on Ubuntu OS. (Ubuntu 14.04+ tested and is recommended). It is important that the graphics card driver is installed properly (the official Ubuntu packages for nVidia and AMD(ATI) graphics cards are available). grSim will compile and run in both 32 and 64 bits Linux and Mac OS.
6 
7 GrSim is written in C++, in order to compile it, you will need a working toolchain and a c++ compiler.
8 
9 ## Dependencies
10 
11 GrSim depends on:
12 
13 - [CMake](https://cmake.org/) version 2.8+
14 - [OpenGL](https://www.opengl.org)
15 - [Qt4 Development Libraries](https://www.qt.io) version 4.8+
16 - [Open Dynamics Engine (ODE)](http://www.ode.org)
17 - [VarTypes Library](https://github.com/szi/vartypes)
18 - [Google Protobuf](https://github.com/google/protobuf)
19 - [Boost development libraries](http://www.boost.org/) (needed by VarTypes)
20 
21 **Note:** It's necessary to compile ODE in double precision. This is default when installing the ODE binaries in Ubuntu. However, if you are compiling ODE from source (e.g on Mac OS), please make sure to enable the double precision during the configuration step: `./configure --enable-double-precision`.
22 
23 ### Linux/Unix Installation
24 
25 If you run a Debian system, or derivative, first ensure that these dependencies are there:
26 
27 ```bash
28 $ sudo apt-get install git build-essential cmake libqt4-dev libgl1-mesa-dev libglu1-mesa-dev libprotobuf-dev protobuf-compiler libode-dev libboost-dev
29 ```
30 
31 Next compile and install VarTypes from source. In the following we install VarTypes from source using `git`.
32 
33 ```bash
34 $ cd /tmp
35 $ git clone https://github.com/szi/vartypes.git
36 $ cd vartypes
37 $ mkdir build
38 $ cd build
39 $ cmake ..
40 $ make
41 $ sudo make install
42 ```
43 
44 Next, clone grSim into your preferred location.
45 
46 ```bash
47 $ cd /path/to/grsim_ws
48 $ git clone https://github.com/mani-monaj/grSim.git
49 $ cd grSim
50 ```
51 
52 Create a build directory within the project (this is ignored by .gitignore):
53 
54 ```bash
55 $ mkdir build
56 $ cd build
57 ```
58 
59 Run CMake to generate the makefiles:
60 
61 ```bash
62 $ cmake ..
63 ```
64 
65 Then compile the program:
66 
67 ```bash
68 $ make
69 ```
70 
71 The binary is copied to the `../bin` folder after a successful compilation.
72 
73 ### Mac OS X Installation
74 
75 Pre-requirements:
76 
77 - [Xcode](https://developer.apple.com/xcode/) or Xcode Command Line Tools 8.0 or newer
78 - [Homebrew](http://brew.sh/) package manager.
79 
80 First ensure the dependencies are there:
81 
82 ```bash
83 $ brew install cmake
84 $ brew install ode --with-double-precision
85 $ brew tap FreeCAD/freecad
86 $ brew install freecad/freecad/qt
87 $ brew install protobuf
88 ```
89 
90 If you run into build issues, you may need to run this first:
91 
92 ```bash
93 $ brew update
94 $ brew doctor
95 ```
96 
97 Next we need to install VarTypes manually. Please refer to the documentation above for the procedure.
98 
99 **IMPORTANT NOTE:** VarTypes currently does not compile on MacOS. Please use the following fork of Vartypes instead of the official repository: `https://github.com/lordhippo/vartypes.git (branch: osx-fix)`
100 
101 The steps to compile grSim on Mac OS is similar to the steps outlines above for Linux:
102 
103 
104 ```bash
105 $ cd /path/to/grsim_ws
106 $ git clone https://github.com/mani-monaj/grSim.git
107 $ cd ./grSim
108 $ mkdir build
109 $ cd build
110 $ cmake ..
111 $ make
112 ```
113 
114 The binary files (grSim and the sample client) will be placed in `../bin`.
115 
116 ## Notes on the performance
117 
118 When running grSim, check the FPS in the status bar. If it is running at **60 FPS** or higher, everything is ok. Otherwise check the graphics card's driver installation and OpenGL settings.
