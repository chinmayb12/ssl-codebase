30 ## Software Requirements
31  * g++
32  * QT >= 4.3 with opengl and networking support
33  * cmake
34  * Eigen3
35  * Google protocol buffers (protoc)
36  * OpenGL
37  * GLU
38  * libdc1394 Version >= 2.0
39  * libjpeg
40  * libpng
41  * video for linux 2 (v4l)
42 
43 To get all of these packages in (k)ubuntu, run the `InstallPackagesUbuntu.sh` script.
44 
45 Or, in archlinux, run the `InstallPackagesArch.sh` script.

47 ## Hardware Requirements
48  * The system supports 1394B / Firewire 800, but it's also backward compatible with 1394A.
49  * The system also supports basic usb cameras via the [http://linuxtv.org/downloads/v4l-dvb-apis/](Video for Linux (V4L)) drivers. This implementation has only been tested on linux.
50 
51 ### (Optional) Matrix-Vision [BlueFox2](http://www.matrix-vision.com/USB2.0-single-board-camera-mvbluefox-mlc.html)
52 This camera type is supported via the mvIMPACT_acquire library. Tested with mvBlueFOX-MLC200wC. If the library is not found on your computer support for this camera will be disabled silently. To enable the BlueFox2 capture module please go to the Matrix-Vision [driver page](http://www.matrix-vision.com/software-drivers-en.html). Go to Linux => mvBlueFOX (USB2.0). Download the install_mvBlueFOX script and the correct .tgz file for your machine. Open a terminal and navigate to your download folder. For a quick installation run:
53 ```
54 chmod +x install_mvBlueFOX.sh
55 ./install_mvBlueFOX.sh -u
56 ```

65 ## Running
66  1. depending on your OS, you might need to ensure that you
67  have full access to the firewire devices /dev/fw*
68  This *might* require logging in as root.
69  2. alternatively, the V4L drivers will attempt to open a
70  device enumerated as /dev/videoN where N is a zero-based index
71  3. run the software using the following command:
72 ```
73  ./bin/vision
74 ```

76 ### Starting to Capture and Setting Parameters
77  Once the software is running, you should see two empty capture frames
78  on the right, and a data-tree structure on the left. In this
79  data-structure you can setup your camera parameters,
80  such as resolution, capture mode, etc.
81 
82  *A quick hint: the text-field below the data-tree allows for
83  fast search through the data-tree.*
84 
85  See the section of DC1394 parameters below to get an idea of what the
86  parameters do.
87 
88  Once you have them set up, you can start capturing by clicking
89  *"Image Capture/Capture Control/Start"* in the data-tree.
90 
91 #### DC1394 Parameters
92  * If you expand the tree then the capture parameters are in
93  "Image Capture/DC1394/Capture Settings"
94 
95  * Furthermore, conversion settings are in
96  "Image Capture/DC1394/Conversion Settings"
97 
98  * "convert to mode" should currently be "yuv422"
99  for best performance, "capture mode" should also be "yuv422",
100  but can also be a different format such as "yuv411" or "rgb"
101 
102  * Alternatively, if you desire, you can do de-bayering in software,
103  but this will take extra CPU cycles. To do so, you would set the
104  capture mode to e.g. "raw8" and the convert mode to "rgb". Finally,
105  you will need to set "de-bayer" to true, and select the correct
106  de-bayer pattern and desired method.
107 
108  * Capturing supports both DCAM native modes and Format7 modes.
109  This is selected in the "capture format" field. Leaving it on
110  "auto" will attempt native mode first, then `format7_0`.
111 
112  * By default, ISO800 support is disabled. To enable it, mark the
113  field "use ISO800" as true.
114 
115 ##### ONLINE PARAMETERS
116  * Once you start capturing, you should see the realtime video image
117  on the right. Furthermore, in the data tree, you should be able to go to the
118  "Camera Parameters" node which will then be expandable and show
119  all of your DCAM parameters.
120 
121  * You can adjust all of these parameters in real-time.
122 
123  * Note that the read-out of these parameters from the camera
124  only happens automatically if "auto refresh params" in the
125  Capture Control is set to true. Otherwise, you can use
126  the "re-read params" "Refresh" button to refresh them manually.
127  For performance reasons, it might make sense to set auto refresh
128  to false, so the bus is not being flooded with too much control
129  data and has full bandwidth available for the video streaming.
130 
131 #### Storage of Settings and Parameters
132 
133  * When you quit the application normally *parameters will not be
134  automatically saved*. Insetad, you need to click on the `Save Settings`
135  button immediately under the main `Vision System` node.
136 
137  * All settings should be automatically restored during the next
138  program start.
139 
140  * In case the files should ever become corrupted, or the
141  program refuses to start completely when parsing the XML files
142  (this should normally never occur) then simply delete all
143  XML files and restart. The program should restore its default
144  settings.
